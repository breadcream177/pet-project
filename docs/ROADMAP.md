# 챙겨펫 1차 배포 로드맵

## 방향

1차 배포는 모바일 중심 웹 앱으로 진행합니다. 사용자가 휴대폰에서 오늘 챙길 일을 빠르게 확인하고 완료 체크하는 경험을 우선합니다.

## MVP 범위

1. 모바일 중심 앱 레이아웃
2. 로그인, 회원가입, 로그아웃
3. 사용자별 반려동물 등록
4. 오늘 일정 등록
5. 오늘 할 일 완료 체크
6. 반복 일정 MVP
7. PWA 설치 지원
8. Vercel 배포

## 구현 순서

### 1. 모바일 UI 기반 정리

- 하단 네비게이션 검토
- 터치하기 쉬운 버튼 크기
- 오늘 할 일 중심 홈 화면
- 작은 화면에서 입력 폼 사용성 확인

### 2. 인증

- Supabase 프로젝트 생성
- 이메일/비밀번호 회원가입
- 로그인
- 로그아웃
- 인증 상태에 따른 접근 제어

### 3. 반려동물 저장

- 반려동물 테이블 설계
- 사용자별 데이터 분리
- 생성, 조회, 수정, 삭제

### 4. 일정

- 일정 테이블 설계
- 반려동물별 일정 생성
- 오늘 일정 조회
- 완료 체크

### 5. 반복 일정

- 매일, 매주 반복 MVP
- 오늘 할 일 생성 기준 정리
- 과도한 데이터 생성 방지

### 6. PWA와 배포

- manifest 설정
- 모바일 아이콘
- 기본 캐시 정책
- Vercel 배포
- 환경 변수 설정

## 보안 기준

- 비밀번호 직접 저장 금지
- Supabase Auth 사용
- 사용자별 Row Level Security 적용
- 클라이언트에 민감 정보 노출 금지
- `.env` 파일 Git 커밋 금지
- 개인정보가 로그에 남지 않도록 주의

## 이력서 포인트

- Next.js App Router 기반 모바일 중심 웹 앱
- Supabase Auth와 사용자별 데이터 분리
- TypeScript 기반 데이터 타입 설계
- 반복 일정 및 완료 체크 로직
- PWA와 Vercel 배포
- Git 기반 기능 단위 개발 흐름
